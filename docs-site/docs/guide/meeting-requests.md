# Meeting Requests

Meeting Requests let any team member propose a meeting for Manager or Admin review. Approved requests automatically create a Google Calendar event and send invites.

---

## Submitting a Request

**Available to:** Admin, Manager, Member

1. Navigate to `/meeting-requests`.
2. Click **New Request**.
3. Fill in the form:

| Field | Description |
|---|---|
| **Title** | Subject of the meeting |
| **Agenda** | What needs to be discussed |
| **Proposed Time** | Date and time |
| **Duration** | Length in minutes |
| **Location** | Room name or video conferencing link |
| **Attendees** | Search and add team members |

4. Click **Submit**. The request status shows as **Pending**.

You will receive a notification when your request is approved or rejected.

---

## Reviewing Requests

**Available to:** Admin, Manager only

Managers and Admins see **all** pending requests. Members only see their own.

1. Open `/meeting-requests`.
2. Click a request row to open the detail view.
3. Review the agenda, proposed time, and attendee list.
4. Choose an action:

### Approve

Click **Approve** to:
- Change the request status to **Approved**
- Create a Google Calendar event automatically
- Send calendar invites to all listed attendees

### Reject

1. Click **Reject**.
2. Enter **Review Notes** explaining the reason (e.g., "Use async Loom instead" or "Please reschedule to next week").
3. Confirm the rejection.

The requester is notified with your notes.

---

## Request Statuses

| Status | Meaning |
|---|---|
| **Pending** | Awaiting Manager or Admin review |
| **Approved** | Calendar event created, invites sent |
| **Rejected** | Request declined — see review notes |

---

## Tips

- Include a clear agenda. Requests with no agenda are harder to approve quickly.
- If a request is rejected, you can submit a revised version with the feedback addressed.
- Managers: use review notes even on approvals to confirm any changed time or location.
